命名規則に従う理由

命名規則とは、変数名などコンピュータープログラムのソースコード上で開発者が名づける識別名についての決まり事。
class名やid名、関数名などプログラム上で定義して扱う対象に名前をつける際に複数人で開発を行う場合などに、
各人が各々で異なる考え方、基準で名前をつけてしまうと名前の衝突や使いまわしなどによりバグが生じる原因となる。
可読性や視認性が損なわれ、開発効率や保守性の悪化原因ともなるので注意が必要。
よく定義された命名規則に従うと、対象の意味や機能、使い方などが名前からある程度推測することができるようになり、
コード自体がドキュメントの役割を果たすようになる。
プログラミングでよく使う英単語のまとめ記事

ケバブケースを使用したclass名

class = "put-name"---ハイフンを使用して文字を区切る名称

スネークケースを使用したclass名

class = "put_name"---アンダーバーを使用して文字を区切る名称

キャメルケースを使用したclass名

class = "putName"---JavaScriptの関数名、変数名の命名に使用。
２ワード構成の命名を行う場合、２ワード目の頭文字を大文字にする。
