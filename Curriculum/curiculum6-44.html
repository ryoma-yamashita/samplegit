Sassとは

「Syntactically(文法的に) Awesome(イケてる・すごい) StyleSheet(スタイルシート)」の略。
Sassはハンプソン・キャトリン氏が設計し、ネイサン・バイゼンバウム氏が開発した、CSSのメタ言語になる。
メタ言語とは、ある言語について何らかの記述をするための言語で、特定のルールを追加し具体的な応用を可能にするもの。
つまりSassは、「CSSを拡張し、文字通りイケてる書き方ができるようにしたもの」ということになる。

Sassのメリット

規模の大きなサイトになってくると、どうしてもコードが複雑化してくる。そこで、コードの量が増えすぎてしまうことを防ぎ、メンテナンス性を高めようとSassが作られた。
Sassの記述方式にはSASS記法とSCSS記法がある。
SASS記法は、波括弧｛｝やコロン：、セミコロン；などが必要なく、効率的にコードを書けるもので、.sassという拡張子を使う。
SCSS記法は、今までのCSSと互換性を持たせるために後から追加された記法で拡張子は.scssとなる。
SCSS記法では、通常のCSSがわかる人であれば問題なくコードが理解できるようになっていることもあり、現在はSCSS記法が主流になっている。

  • 入れ子構造が使えて把握がしやすい

    • ・・・divの子要素の中にh2とbuttonがある場合、div、h2、buttonそれぞれにCSSを当てたいとき、通常の記法であれば、
    div {} div h2{} div button {}というようにそれぞれを指定する必要があるが、div { h2 {} button{} }のように
    入れ子構造で書くことが可能で、コードの可読性が向上する。さらに、&を使用することでセレクタの親要素を取得することができる
    これはホバー時のスタイルを指定するときや、命名規則で親セレクタを取得するときに便利。
    例）mainとmain_contentのクラスがあった場合.main { &_content {} }と記述することが可能、SASS記法の場合は{}を省略することが可能。

  • 四則演算ができる

    • ・・・JavaScriptのように四則演算が可能。CSSでもcalcを使用することで演算が可能ではあるが、Sassの場合は後述する変数と組み合わせることでより便利に使うことが可能
    例）$multiple: 8px;(multipleを8pxと定義)＿＿.main { width: $multiple * 5 }(mainのwidthは8*5=40pxとすることが可能)

  • 変数が使える

    • ・・・SCSS記法での記述の場合、＄マークを使用することで変数を指定することが可能。共通の数値や色の指定などに非常に便利。
    ただ変数の定義は、通常のCSSでも使用可能となり、Sass独自のメリットとは言いづらくなってきている。JavaScriptと同様、グローバル変数とローカル変数がある。
    グローバル変数とは、そのファイル全体で使うことのできる変数。対してローカル変数は、定義された{}内でのみで使用することができる変数。

  • ファイルを分割できる

    • ・・・Webサイトが複雑化してくると、CSSの記述量が増えてくる。Sassはファイルを分割し、インポートすることで解決する。
    例えば、変数をまとめたファイルを読み込むなどの処理をすることで、実装している機能ごとにCSSを分けることが可能。
    また、後述するMixin（ミックスイン）や継承で用いるコードを分割しておくということにも応用が可能。
    ファイルの冒頭に_（アンダーバー）を入れることで、そのファイルをパーシャル化（CSSとして書き出さないように設定）することが可能。
    例えば、変数をまとめたファイル名を_variables.scssなどとし、importすることでコード管理が簡易化することが可能。

  • Mixin（ミックスイン）でコードを引用できる

    • ・・・Mixinは、スタイルに引数を持たせることで、そのコードの一部書き換えて表示させることが可能。
    まずはMixinを作成する。作成するには"@mixin ミックスインの名前(引数){}"と記載する。
    例）$multiple: 8px;＿＿@mixin headings($height) { font-size: $multiple * $height; margin-bottom: $multiple * ($height - 2);}
    headings(Mixinの名前)、$height(引数)、上記の場合font-sizeは8*$height(のちに指定して使用)px、margin-bottomは8*($height-2)pxと指定しておく
    次に、作成したMixinを@include ミックスインの名前(引数)とすることで読み込むことが可能。
    例）h2 { @include headings(6); }＿＿h3 { @include headings(5); }この場合は、$heightに指定した値が代入されるので、
    h2のスタイルはfont-sizeが8*6=48px、margin-bottomが8*(6-2)=32pxとなり、h3のスタイルはfont-sizeが8*5=40px、margin-bottomが8*(5-2)=24pxとなる。
    上記のようにすることで、コードの記述量が減り、メンテナンスも楽になることで保守性を高めることが可能。

  • コードを継承で使いまわせる

    • Mixinと似ているが、引数をもたず、コードをそのまま使いまわすようなイメージ。Mixinと同様に継承元を作成し読み込むということに変わりはない。
    まずは継承元を作成する。継承では、もともと記述してあるスタイルをそのまま読み込む。
    例）.block { display: block; margin: 0 auto; }というスタイルを用意しておく。
    次に、用意したスタイルを@extendを使い継承させる。
    例）div { @extend .block }と記述することで、div { display: block; margin: 0 auto; }と同じようにスタイルが適用される。
    継承では、既存のclassを読み込むことができるが、継承元をCSSで書き出さないようにする（パーシャル化する）ことが可能。それには%の後に名前をつける。
    例）%.block { display: block; margin: 0 auto; }＿＿div { @extend %.block }と記述するとdivにはスタイルが適用されるが、継承元である.blockのスタイルが読み込まれることはない。
    Mixinと継承のどちらも似たような機能を持っているので、どちらを使っても実装が可能だが、明確な使い分けのルールがあるわけではない。
    引数の値によってスタイルを変える場合は、Mixinを用いることが適している。反対に、引数を持たない場合や、スタイルとして完成しているものを踏襲したスタイルを設定する場合には継承を使うのが適している。

  • JavaScriptのように関数が使える

    • JavaScriptと同様に配列を操作するようなものや、for文、if文、色の三要素(色相・明度・彩度)に関するものがある。他にも自作で関数を設定することができる。
    if文は、Mixinと組み合わせて使われることが多い。
    例）@mixin headings($height, $space: true) {font-size: $multiple * $height; @if $space {margin-bottom: $multiple * ($height - 2);}}
    この場合$spaceがtrueの時にmargin-bottomが適用される、余白が必要のない時は、@include headings(5, fales)とすることでfont-sizeのみ適用させることができる。
    例）$color: #c81c14;＿＿.div1 { background: $color; }
    .div2 { background: adjust-hue($color, 60deg);}＿＿色相の角度を変更＿＿.div2 { backgroun: #c0c814; }となる。
    .div3 { background: grayscale($color); }＿＿モノトーンに変更＿＿.div3 { background: #6e6e6e; }となる。
    .div4 { background: staturate($color, 10%); }＿＿彩度を指定した割合分上げる＿＿.div4 { background: #d31209; }となる。
    .div5 { background: desaturate($color, 15%); }＿＿彩度を指定した割合分下げる＿＿.div5 { background: #b82b25; }となる。
    .div6 { background: lighten($color, 5%);}＿＿明度を指定した割合分上げる＿＿.div6 { background: #df1d16; }となる。
    .div7 { background: darken($color, 20%);}＿＿明度を指定した割合分下げる＿＿.div7 { background: #6b0f0b; }となる。
    .div8 { background: rgba($color, 0.5);}＿＿透明度（opacity）を変更する＿＿.div8 { background: rgba(200, 28, 20, 0.5); }となる。

Sassのデメリット

Sassは、SCSS記法であれば、そこまで学習コストが高くない。CSSが書ければその延長ですぐに利用が可能。他に考えられる懸念のひとつは、
Sassを使うための環境を整えるのが面倒。Sassを使うには、Sassと一緒にRubyやコンパイラをインストールするか、Gulpを導入する必要がある。
しかし最近では、JavaScriptをそのまま使うということは少なく、Vue.jsやNuxt.jsなどのフレームワークを使用することが多い。
Vue.jsやNuxt.jsではSassを簡単に使えるようにするモジュール（特定の機能を持った部品）が用意されているので、このデメリットは比較的カバーしやすい。
もうひとつは、CSSを使える人が全員Sassを使っているわけではないということ。自分の所属する組織、または制作物を納品するタイプ仕事の場合のクライアントに
導入のない場合、上司の説得、納品が困難である場合がある。しかし、CSSのメタ言語の中でSassは代表的なもののひとつなので、個人のスキルとして使えて損はない。

Gulpとは

Node.jsのパッケージ（プラグイン）として開発されているWeb開発を効率的に行うためのツール。様々な処理や作業を自動化することができ実際の開発現場でも使用されている。
Gulpを使うと、ファイルを圧縮してファイルサイズを軽くしたり、サーバーにアップロードをする作業を自動化したりすることで開発を効率的に行うことが可能。
Gulpに似ているツールは他にもあるが、Gulpは処理速度が早く、併用して使えるNode.jsのパッケージが豊富なことから、多くの人に使用されている。
Gulpの一番の特徴はサイトのトップページで「ストリーミングビルドシステム」と自ら名乗っているように、ファイルの処理をストリームで行うというところ。この特徴によって複雑なタスクも細かくカスタマイズして書くことが可能。

コンパイルとは

特定のプログラミング言語を用いて記述されたコンピュータープログラム（ソースコード）をコンピューターが直接実行可能な形式のプログラム（オブジェクトコード）に変換すること。コンパイラーはこのタスクを実行するソフトウェア。
ソースコードは人間に読み書きしやすい形式の言語によって記述されたもので、コンピューター内部のマイクロプロセッサ（MPU/CPU）にそのまま与えても実行はできず、コンピューターの理解できる機械語のプログラムへ変換しなければならない。
この変換作業を一括して行うことをコンパイルと呼ぶ。今日のソフトウェア開発ではコンパイルだけで起動可能なプログラムを得られることは少なく、コンパイル直前に前処理（プリプロセス）が必要であったり、コンパイル済みのプログラムを起動可能な状態にするために
他のプログラムと連結（リンク）させたりする工程を経ることが多い。コンパイルを含むこうした一連の工程をビルド（build）あるいはメイク（make）という。
機械語への返還にはコンパイル以外の手法もあり、実行時にソースコードを少しずつ読み込んで変換しながら同時に実行する方式などがあり、スクリプト言語などでよく利用されている。

Gulpを使う理由

Gulpを使うと、ファイルを圧縮してファイルサイズを軽くしたり、サーバーにアップロードをする作業を自動化したりすることで開発を効率的に行うことが可能。
Gulpに似ているツールは他にもあるが、Gulpは処理速度が早く、併用して使えるNode.jsのパッケージが豊富なことから、多くの人に使用されている。
Gulpはビルドシステムヘルパーであり、類似したものにタスクランナーであるGruntがある。どちらもいろいろな機能（プラグイン）を使い、目的とする成果物を求める一連の処理（タスク）を実行しているがアプローチ方法が異なる。
Gruntの場合、タスク＝プラグインといった感じに、その単一のタスク（プラグイン）を実行させるのが主な役割であり、単一のタスクをうまいこと組み合わせて実行することで、ビルドという大きな目標にたどり着くというアプローチをとる。
Gulpの場合、プラグインはあくまでも入力に対して処理を行うためのものであり、GulpのタスクはGruntでいうところのビルドに近いものである。タスクの中で様々なプラグインを組み合わせることでタスク＝ビルドを行うといったアプローチをとる。
プラグインを容易に組み合わせられるので、柔軟なタスクを作成することが可能であるとともに、タスク同士の依存関係を極力減らすことが可能。
